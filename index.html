Git

  • git status: checks what branches we are currently on
  • gitcheck out -b branch name: creates a new branch and switches to it
  • Repository (repo): A collection of files and thier revision history.
  • Clone: Saving changes to files in a repository with a message describing the changes.
  • Commit: A way to save changes to files in the repository with a message decribing changes.
  • Branch: A parallel version of a repo allowing for separate development paths.
  • Merge: Combining changes from different branches into one.
  • Remote: A version of the repo stored on a server, often used for collaoration.
  • Pull Request (PR): A reguest to merge changes from one branch into another, commonly used in collaborative development.
  • Push: Sending committed changes from your local repo to a remote repo.
